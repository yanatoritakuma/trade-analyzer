# 画面devspec - 新規登録画面

**画面ID**: SCR-02  
**パス**: `/register`  
**作成日**: 2026-06-06  
**バージョン**: 1.0

---

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 目的 | 招待コードを持つユーザーが新規アカウントを作成する |
| 対象ユーザー | 未認証ユーザー（招待コード保持者のみ登録可能） |
| 認証要否 | 不要 |
| ログイン済みの場合 | `/`（ダッシュボード）へリダイレクト |

---

## 2. レイアウト・コンポーネント構成

```
┌─────────────────────────────────┐
│         [ロゴ / サービス名]      │
│                                 │
│  ┌───────────────────────────┐  │
│  │    招待コード               │  │
│  │  [input: text]            │  │

│  │                           │  │

│  │  [input: password] [👁]   │  │
│  │                           │  │
```

### コンポーネント一覧

| コンポーネント | 種類 | 説明 |
|-------------|------|------|
| 招待コード入力 | `<input type="text">` | 大文字変換・オートフォーカス |
| お名前入力 | `<input type="text">` | 表示名 |
| メールアドレス入力 | `<input type="email">` | - |
| パスワード入力 | `<input type="password">` | 表示/非表示トグルあり |
| パスワード確認入力 | `<input type="password">` | 表示/非表示トグルあり |
| 登録するボタン | `<button type="submit">` | 処理中はローディング表示 |
| ログインリンク | `<a href="/login">` | ログイン画面へ遷移 |

## 3. 使用API

| メソッド | エンドポイント | タイミング |
|---------|-------------|---------|
| POST | `/api/auth/register` | 登録するボタン押下時 |

### リクエスト

```json
{
  "invitation_code": "TRADE-XXXX-XXXX",
  "name": "山田太郎",
  "email": "user@example.com",
  "password": "password123"
}
```

### レスポンス（成功）

```json
{
  "message": "アカウントを作成しました",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "name": "山田太郎",
    "role": "user"
  }
}
```

### レスポンス（失敗）

```json
{ "error": "無効な招待コードです" }
{ "error": "このメールアドレスはすでに使用されています" }
```

## 4. ユーザー操作と挙動

### 4.1 正常系

| 操作 | 挙動 |
|------|------|
| 画面表示 | 招待コード入力欄にオートフォーカス |
| 登録するボタン押下 | バリデーション → API呼び出し → ログイン画面へリダイレクト |
| 登録成功 | `/login` へリダイレクト・「登録が完了しました。ログインしてください」のメッセージ表示 |
| ログインリンク押下 | `/login` へ遷移 |

### 4.2 異常系

| 状況 | 挙動 |
|------|------|
| 必須未入力で登録押下 | 各フィールドにバリデーションエラーを表示 |
| 招待コード無効 | 「無効な招待コードです」を表示 |
| 招待コード期限切れ | 「招待コードの有効期限が切れています」を表示 |
| 招待コード使用済み | 「この招待コードはすでに使用されています」を表示 |
| メール重複 | 「このメールアドレスはすでに使用されています」を表示 |
| パスワード不一致 | 「パスワードが一致しません」を表示 |
| ネットワークエラー | 「通信エラーが発生しました」を表示 |
| API呼び出し中に再押下 | ボタンをdisabledにして二重送信を防止 |

## 5. バリデーション

| フィールド | ルール | エラーメッセージ |
|-----------|--------|--------------|
| 招待コード | 必須 | 「招待コードを入力してください」 |
| お名前 | 必須 | 「お名前を入力してください」 |
| お名前 | 最大50文字 | 「お名前は50文字以内で入力してください」 |
| メールアドレス | 必須 | 「メールアドレスを入力してください」 |
| メールアドレス | メール形式 | 「正しいメールアドレスを入力してください」 |
| パスワード | 必須 | 「パスワードを入力してください」 |
| パスワード | 8文字以上 | 「パスワードは8文字以上で入力してください」 |
| パスワード | 英字・数字を各1文字以上 | 「パスワードは英字と数字を含めてください」 |
| パスワード確認 | 必須 | 「パスワード（確認）を入力してください」 |
| パスワード確認 | パスワードと一致 | 「パスワードが一致しません」 |

---

## 6. ローディング・状態管理

| 状態 | UIの変化 |
|------|---------|
| 初期表示 | 通常表示 |
| API呼び出し中 | ボタンを「登録中...」に変更・disabled |
| 成功 | `/login` へリダイレクト |
| 失敗 | エラーメッセージ表示・ボタンを元に戻す |

---

## 7. 画面遷移

```
/register
  ├── 登録成功 → /login（成功メッセージ付き）
  └── ログインリンク → /login
```

## 8. 実装メモ（Next.js）

```typescript
// ファイル構成
app/
  └── register/
      └── page.tsx

// Zodスキーマ
const registerSchema = z.object({
  invitationCode: z.string().min(1, '招待コードを入力してください'),
  name: z.string().min(1, 'お名前を入力してください').max(50),
  email: z.string().min(1).email('正しいメールアドレスを入力してください'),
  password: z.string().min(8, '8文字以上').regex(/^(?=.*[a-zA-Z])(?=.*[0-9])/, '英字と数字を含めてください'),
  passwordConfirm: z.string().min(1, 'パスワード（確認）を入力してください'),
}).refine(data => data.password === data.passwordConfirm, {
  message: 'パスワードが一致しません',
  path: ['passwordConfirm'],
});
```
