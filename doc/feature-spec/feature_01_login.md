# 画面devspec - ログイン画面

**画面ID**: SCR-01  
**パス**: `/login`  
**作成日**: 2026-06-06  
**バージョン**: 1.0

---

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 目的 | 登録済みユーザーがメールアドレスとパスワードでログインする |
| 対象ユーザー | 未認証の全ユーザー（admin / user 共通） |
| 認証要否 | 不要 |
| ログイン済みの場合 | `/`（ダッシュボード）へリダイレクト |

---

## 2. レイアウト・コンポーネント構成

```
┌─────────────────────────────────┐
│                                 │
│         [ロゴ / サービス名]      │
│                                 │
│  ┌───────────────────────────┐  │
│  │    メールアドレス           │  │
│  │  [input: email]           │  │
│  │                           │  │
│  │    パスワード               │  │
│  │  [input: password] [👁]   │  │
│  │                           │  │
│  │  [ログインボタン]           │  │
│  │                           │  │
│  │  アカウントをお持ちでない方   │  │
│  │  → [新規登録はこちら]       │  │
│  └───────────────────────────┘  │
│                                 │
└─────────────────────────────────┘
```

### コンポーネント一覧

| コンポーネント | 種類 | 説明 |
|-------------|------|------|
| ロゴ / サービス名 | テキスト or 画像 | サービス名を表示 |
| メールアドレス入力 | `<input type="email">` | オートフォーカス |
| パスワード入力 | `<input type="password">` | 表示/非表示トグルあり |
| パスワード表示トグル | アイコンボタン（👁） | 押すと平文/マスク切り替え |
| ログインボタン | `<button type="submit">` | 処理中はローディング表示 |
| 新規登録リンク | `<a href="/register">` | 登録画面へ遷移 |
| エラーメッセージ | インラインテキスト | 入力欄の下またはフォーム上部 |

---

## 3. 使用API

| メソッド | エンドポイント | タイミング |
|---------|-------------|---------|
| POST | `/api/auth/login` | ログインボタン押下時 |

### リクエスト

```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

### レスポンス（成功）

```json
{
  "message": "ログインしました",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "name": "山田太郎",
    "role": "user"
  }
}
```

※ JWTトークンはレスポンスボディではなく **HttpOnly Cookie** にセットされる。

### レスポンス（失敗）

```json
{
  "error": "メールアドレスまたはパスワードが正しくありません"
}
```

## 4. ユーザー操作と挙動

### 4.1 正常系

| 操作 | 挙動 |
|------|------|
| 画面表示 | メールアドレス入力欄にオートフォーカス |
| ログインボタン押下 | バリデーション実行 → API呼び出し → リダイレクト |
| ログイン成功（role: user） | `/`（ダッシュボード）へリダイレクト |
| ログイン成功（role: admin） | `/admin`（管理者ダッシュボード）へリダイレクト |
| 新規登録リンク押下 | `/register` へ遷移 |
| パスワードトグル押下 | パスワードの表示/非表示を切り替え |

### 4.2 異常系

| 状況 | 挙動 |
|------|------|
| 未入力でログイン押下 | バリデーションエラーを表示（API呼び出しなし） |
| メール形式が不正 | 「正しいメールアドレスを入力してください」を表示 |
| 認証失敗（メール・PW不一致） | 「メールアドレスまたはパスワードが正しくありません」を表示 |
| アカウント停止済み | 「このアカウントは無効です。管理者にお問い合わせください」を表示 |
| ネットワークエラー | 「通信エラーが発生しました。しばらく経ってから再試行してください」を表示 |
| API呼び出し中に再押下 | ボタンをdisabledにして二重送信を防止 |

## 5. バリデーション

| フィールド | ルール | エラーメッセージ |
|-----------|--------|--------------|
| メールアドレス | 必須 | 「メールアドレスを入力してください」 |
| メールアドレス | メール形式 | 「正しいメールアドレスを入力してください」 |
| パスワード | 必須 | 「パスワードを入力してください」 |
| パスワード | 1文字以上 | 「パスワードを入力してください」 |

※ パスワードの文字数・複雑さチェックはログイン時は行わない（登録時のみ）

---

## 6. ローディング・状態管理

| 状態 | UIの変化 |
|------|---------|
| 初期表示 | 通常表示 |
| API呼び出し中 | ログインボタンを「ログイン中...」に変更・disabled |
| 成功 | リダイレクト |
| 失敗 | エラーメッセージ表示・ボタンを元に戻す |

---

## 7. セキュリティ考慮事項

| 項目 | 対応 |
|------|------|
| 認証失敗メッセージ | メール・パスワードのどちらが間違いか特定できないメッセージにする |
| パスワードの取り扱い | 画面上でマスク表示・ログに出力しない |
| ログイン済みアクセス | middleware.tsでトークンを確認し `/` にリダイレクト |
| CSRF対策 | HttpOnly Cookie + SameSite=Strict で対応 |

---

## 8. 画面遷移

```
（未認証状態で保護画面にアクセス）
    → /login にリダイレクト

/login
  ├── ログイン成功（user）  → /
  ├── ログイン成功（admin） → /admin
  └── 新規登録リンク       → /register
```
