# SCR-10 ユーザー管理（/admin/users）

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 目的 | 登録ユーザーの一覧表示・停止・復活・削除 |
| 対象ユーザー | **admin のみ** |
| 認証要否 | 必要（admin権限） |

## 2. レイアウト

```
┌─────────────────────────────────────────┐
│  ユーザー管理                             │
│                                         │
│  ┌────┬──────────────┬─────┬──────┬───┐ │
│  │ID  │メール         │名前 │状態  │操作│ │
│  ├────┼──────────────┼─────┼──────┼───┤ │
│  │1   │admin@...     │管理者│有効  │ - │ │
│  │2   │user@...      │山田  │有効  │[停止][削除]│
│  │3   │user2@...     │鈴木  │停止中│[復活][削除]│
│  └────┴──────────────┴─────┴──────┴───┘ │
└─────────────────────────────────────────┘
```

## 3. 使用API

| メソッド | エンドポイント | タイミング |
|---------|-------------|---------|
| GET | `/api/admin/users` | 画面表示時 |
| PATCH | `/api/admin/users/:id` | 停止・復活ボタン押下時 |
| DELETE | `/api/admin/users/:id` | 削除ボタン押下時 |

## 4. ユーザー操作と挙動

| 操作 | 挙動 |
|------|------|
| 「停止」ボタン | 確認ダイアログ → `PATCH`（is_active: false） → 状態を「停止中」に更新 |
| 「復活」ボタン | 確認ダイアログ → `PATCH`（is_active: true） → 状態を「有効」に更新 |
| 「削除」ボタン | 確認ダイアログ（「削除すると元に戻せません」） → `DELETE` → 一覧から削除 |

## 5. 制約

- adminユーザー自身は停止・削除ボタンを表示しない
- 削除は論理削除ではなく物理削除

# SCR-11 招待コード管理（/admin/invitations）

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 目的 | 招待コードの発行・一覧表示・無効化 |
| 対象ユーザー | **admin のみ** |

```

│  │コード         │有効期限  │状態│操作  ││
│  ├──────────────┼──────────┼────┼──────┤│
│  │TRADE-ABCD-1234│6/13まで │有効│[無効化]││
│  │TRADE-EFGH-5678│6/10まで │使用済│ - ││
│  │TRADE-IJKL-9012│6/08まで │期限切│ - ││

│                              │
│  [キャンセル]  [発行する]      │
└──────────────────────────────┘
```

## 3. 使用API

| メソッド | エンドポイント | タイミング |
|---------|-------------|---------|
| GET | `/api/admin/invitations` | 画面表示時 |
| POST | `/api/admin/invitations` | 「発行する」押下時 |
| DELETE | `/api/admin/invitations/:id` | 「無効化」押下時 |

### POST リクエスト

```json
{ "expires_days": 7 }
```

### POST レスポンス

```json
{
  "code": "TRADE-ABCD-1234",
  "expires_at": "2026-06-13T23:59:59Z"
}
```

## 4. ユーザー操作と挙動

| 操作 | 挙動 |
|------|------|
| 「+ コード発行」押下 | 発行ダイアログを表示 |
| 「発行する」押下 | API呼び出し → 一覧に追加 → コードをクリップボードにコピー |
| 「無効化」押下 | 確認ダイアログ → DELETE → 状態を「無効」に更新 |
| コードをクリック | クリップボードにコピー・「コピーしました」のSnackbar表示 |

## 5. ステータス表示

| ステータス | 表示 | 色 |
|-----------|------|-----|
| 有効 | 「有効」 | 緑 |
| 使用済み | 「使用済」 | グレー |
| 期限切れ | 「期限切」 | オレンジ |
| 無効化済み | 「無効」 | 赤 |

## 5. バリデーション

| フィールド | ルール | エラーメッセージ |
|-----------|--------|--------------|
| テーマ選択 | 1つ以上選択 | 「テーマを1つ以上選択してください」 |
| 時価総額 | 0以上の数値 | 「0以上の数値を入力してください」 |
| 出来高 | 0以上の整数 | 「0以上の整数を入力してください」 |
| 自由プロンプト | 最大1,000文字 | 「1,000文字以内で入力してください」 |
| テーマ名 | 必須・最大100文字・重複不可 | 「テーマ名を入力してください」「すでに存在するテーマ名です」 |

## 6. ユーザー操作と挙動

| 操作 | 挙動 |
|------|------|
| テーマチェックボックス | 複数選択・プレビューに即時反映 |
| 「+ テーマを追加」押下 | テーマ管理タブへ遷移 |
| テーマのドラッグ&ドロップ | 並び順を変更・`PATCH /sort`を呼び出し |
| 「編集」ボタン | 編集ダイアログを表示 |
| 「削除」ボタン | 確認ダイアログ → DELETE |
| 「保存する」押下 | バリデーション → PUT → 「保存しました。次回15:30から反映されます」（緑Snackbar） |

## 3. 使用API

| メソッド | エンドポイント | タイミング |
|---------|-------------|---------|
| GET | `/api/admin/watchlist-candidates` | 画面表示時 |
| PATCH | `/api/admin/watchlist-candidates/:id/approve` | 「承認」押下時 |
| PATCH | `/api/admin/watchlist-candidates/:id/reject` | 「却下」押下時 |

### 承認時の挙動（バックエンド）

```
PATCH /api/admin/watchlist-candidates/:id/approve
  ↓
replace_tickerが指定されている場合
  → watchlistから該当銘柄を削除
候補銘柄をwatchlistに追加
watchlist_candidatesのstatusを 'approved' に更新
```

## 4. ユーザー操作と挙動

| 操作 | 挙動 |
|------|------|
| 「承認」押下 | 確認ダイアログ（「XXXをウォッチリストに追加します。YYYを削除してもよいですか？」）→ 承認API呼び出し → 画面更新 |
| 「却下」押下 | 即座にrejectAPI呼び出し（確認なし）→ 画面から消える |
